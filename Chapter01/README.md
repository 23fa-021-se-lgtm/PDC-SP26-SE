⚡ Concurrency & Parallelism Benchmarks
All three benchmark scripts use the same workload: generating 10,000,000 random floats across 10 workers.
do_something.py — Shared Utility
pythondef do_something(count, out_list):
    for i in range(count):
        out_list.append(random.random())
A simple CPU-bound task that fills a list with random numbers. Used as the target function in all benchmarks.

serial_test.py — Baseline (No Concurrency)
Runs do_something() 10 times sequentially in a single thread.
bashpython serial_test.py
# Output: serial time= X.XX seconds

multithreading_test.py — Threading
Spawns 10 threads using threading.Thread, each running do_something().
bashpython multithreading_test.py
# Output: multithreading time= X.XX seconds

⚠️ Note: Due to Python's Global Interpreter Lock (GIL), threads do not achieve true parallelism for CPU-bound tasks. Performance may be similar to or worse than serial execution.


multiprocessing_test.py — Multiprocessing
Spawns 10 separate processes using multiprocessing.Process, each running do_something().
bashpython multiprocessing_test.py
# Output: multiprocesses time= X.XX seconds

✅ Bypasses the GIL by using separate memory spaces. Achieves true parallelism on multi-core systems.


thread_and_processes.py — Combined Demo
A single script that runs both multithreading and multiprocessing back-to-back, printing timing results for a direct comparison. The serial section is included but commented out.
bashpython thread_and_processes.py

🚀 Getting Started
Prerequisites

Python 3.5+
No external dependencies — uses only the Python standard library (threading, multiprocessing, random, time, os)

Running the Examples
bash# Clone or extract the project
cd Chapter01

# Run any script directly
python classes.py
python flow.py
python serial_test.py
python multithreading_test.py
python multiprocessing_test.py
python thread_and_processes.py

Important: Scripts using multiprocessing must be run as top-level scripts (not from an interactive shell). The if __name__ == "__main__": guard is required on Windows.
